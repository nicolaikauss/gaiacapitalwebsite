# Internationalization (i18n) Guide

This project uses `react-i18next` for internationalization support with English and French languages across the entire application.

## Setup

The i18n configuration is located in `src/i18n/index.ts` and uses:
- `react-i18next` for React integration
- `i18next-browser-languagedetector` for automatic language detection
- Local storage for persisting language preferences
- Automatic document language and title updates

## Adding New Translations

### 1. Add Keys to Translation Files

Add new translation keys to both language files:

**`src/i18n/locales/en.json`**
```json
{
  "dashboard": {
    "newFeature": "New Feature"
  }
}
```

**`src/i18n/locales/fr.json`**
```json
{
  "dashboard": {
    "newFeature": "Nouvelle Fonctionnalité"
  }
}
```

### 2. Use in Components

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <h1>{t('dashboard.newFeature')}</h1>
  );
}
```

## Language Toggle Component

The `LanguageToggle` component is located in `src/components/ui/language-toggle.tsx` and provides:

- Compact EN/FR toggle button
- Automatic language detection and persistence
- Accessibility support with proper ARIA labels
- Tooltip showing current action
- Updates to document language and title

### Usage

```tsx
import { LanguageToggle } from '@/components/ui/language-toggle';

function Header() {
  return (
    <header>
      <LanguageToggle />
    </header>
  );
}
```

### Integration Across All Pages

The language toggle is now integrated into all major pages:
- **Landing Page**: Top-right corner

- **Dashboard**: Header (before logout button)

## Language Detection Order

1. **localStorage** - Previously saved preference
2. **navigator** - Browser language setting
3. **htmlTag** - HTML lang attribute
4. **fallback** - English (en)

## Best Practices

1. **Use descriptive keys**: `dashboard.cards.inventory.title` instead of `title1`
2. **Group related translations**: Keep related strings under the same namespace
3. **Provide fallbacks**: Always include English translations as fallback
4. **Test both languages**: Ensure UI works correctly in both EN and FR
5. **Consider text length**: French text is often longer than English
6. **Consistent placement**: Language toggle appears in top-right corner on all pages
7. **Accessibility**: All translated text maintains proper ARIA labels and screen reader support

## Adding New Languages

To add a new language (e.g., Spanish):

1. Create `src/i18n/locales/es.json`
2. Add Spanish translations
3. Update `src/i18n/index.ts` resources object
4. Update `LanguageToggle` component to support the new language
5. Add tests for the new language

## Troubleshooting

### Common Issues

1. **Missing translations**: Check console for missing key warnings
2. **Language not persisting**: Ensure localStorage is available
3. **Layout shifts**: Use consistent text lengths or CSS to prevent shifts
4. **SEO issues**: Ensure document language attribute is updated

### Debug Mode

Enable debug mode in `src/i18n/index.ts`:
```typescript
i18n.init({
  debug: true, // Enable for development
  // ... other options
});
```
